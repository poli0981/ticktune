# 08 — Internationalization (EN/VI)

Suite 1.0 · 2026-07-21

Vietnamese is the default; English is first-class. Two mechanisms, matched to page
type:

## 1. Static pages — route-based

- `/` = VI, `/en/` = EN mirrors (landing, `legal/*`, 404), following the
  `poli0981.dev` pattern.
- Astro content: one component per page, strings from
  `src/i18n/static/{vi,en}.ts` typed dictionaries; `hreflang` alternates +
  `lang` attribute per route.
- Legal pages: **English is canonical** for interpretation (stated on the VI
  pages); both languages ship at v1.0 — VI translation of legal drafts is a
  roadmap P6 task (`16 §P6`).

## 2. App island — runtime i18next

- `i18next@26`, resources bundled from `src/app/i18n/{en,vi}.json` (no HTTP
  backend, no language detector — explicit toggle only).
- Initial language: persisted setting → else `navigator.language` starts with
  `vi` → VI, else EN.
- Toggle in Z6 header; switch is instant (no reload); persisted to Dexie;
  `document.documentElement.lang` updated (a11y, `03 §8`).

## 3. Key conventions

- Namespace by feature: `setup.*`, `player.*`, `settings.*`, `yt.err.*`,
  `gate.*`, `toast.*`, `finished.*`.
- Keys are stable IDs (`yt.err.blocked.title`), never English sentences.
- Interpolation via i18next `{{count}}`/plurals — VI has no plural forms
  (single form), EN uses `_one/_other`.
- Numbers/dates/durations: `Intl.NumberFormat`/`Intl.DateTimeFormat` with the
  active locale; countdown digits are locale-independent by design.
- `N/A` and `–` fallbacks are **not** translated (universal per portfolio
  convention).
- CI guard: a Vitest test diffs key sets of `en.json` vs `vi.json` — any
  missing/extra key fails the build (`13 §3`).

## 4. Tooling continuity

Dictionaries are plain i18next JSON — directly compatible with the existing
JSONPrism i18next→RESX bridge if strings are ever shared with desktop projects.

## 5. Adding a language later

Post-1.0 JP would follow the portfolio pattern: add `ja.json` + `/ja/` static
routes; no code changes beyond the language menu. Not scheduled for v1.0.
