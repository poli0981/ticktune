# 08 ‚Äî Internationalization (EN/VI)

Suite 1.0 ¬∑ 2026-07-21

Vietnamese is the default; English is first-class. Two mechanisms, matched to page
type:

## 1. Static pages ‚Äî route-based

- `/` = VI, `/en/` = EN mirrors (landing, `legal/*`, 404), following the
  `poli0981.dev` pattern.
- Astro content: one component per page, strings from
  `src/i18n/static/{vi,en}.ts` typed dictionaries; `hreflang` alternates +
  `lang` attribute per route.
- Legal pages: **English is canonical** for interpretation (stated on the VI
  pages); both languages ship at v1.0 ‚Äî VI translation of legal drafts is a
  roadmap P6 task (`16 ¬ßP6`).

## 2. App island ‚Äî runtime i18next

- `i18next@26`, resources bundled from `src/app/i18n/{en,vi}.json` (no HTTP
  backend, **no detector plugin** ‚Äî a one-time `navigator.language` sniff at
  init, described below, and an explicit toggle after that).
- Initial language: persisted setting ‚Üí else `navigator.language` starts with
  `vi` ‚Üí VI, else EN. Implemented once, in `initialLang`
  (`tt-settings-schema.ts`), and the runtime is *told* the result rather than
  deciding it again ‚Äî two mechanisms answering the same question is how they
  come to disagree.
- Toggle in Z6 header; switch is instant (no reload); persisted to Dexie;
  `document.documentElement.lang` updated (a11y, `03 ¬ß8`).

### 2.1 Runtime settings, stated rather than left to the implementer

Filed 2026-07-23, closing the üü° `AUDIT-BACKLOG` finding that named these as
undefined. Both are load-bearing, not defaults worth inheriting:

| Setting | Value | Why |
|---------|-------|-----|
| `interpolation.escapeValue` | **`false`** | `09 ¬ß4` bans `{@html}` and Svelte already escapes every interpolation. i18next's default `true` would escape a second time, so a track called *Bird & Boy* renders as `Bird &amp;amp; Boy` ‚Äî and every interpolated value here is a **track title**, which is user-supplied text |
| `fallbackLng` | **`'en'`** | Not `vi`, even though VI is the UI default. The fallback is the **reference** dictionary: `TtKey` is derived from `en.json`, so EN is the only file guaranteed to hold every key. A key missing from VI renders its English text rather than a raw id |
| missing key | renders the **EN value**, never the id | The key-diff guard (`¬ß3`, `13 ¬ß1`) is build-failing, so this should be unreachable; it defines what it looks like if it ever is. `returnNull: false` keeps a null value from reaching the DOM as the string `"null"` |

‚öÝÔ∏è **`escapeValue: false` is not a style choice and must not be "hardened" back.**
Turning it on does not add safety ‚Äî Svelte's escaping is what provides that ‚Äî it
only corrupts non-ASCII and punctuation in exactly the strings this app is built
to display.

## 3. Key conventions

- Namespace by feature: `setup.*`, `player.*`, `settings.*`, `yt.err.*`,
  `gate.*`, `toast.*`, `finished.*`. The track-info modal (`02 ¬ß8`) files under
  **`player.trackinfo.*`** rather than a namespace of its own.
- Keys are stable IDs (`yt.err.blocked.title`), never English sentences.
- Interpolation via i18next `{{count}}`/plurals ‚Äî VI has no plural forms
  (single form), EN uses `_one/_other`.
- Numbers/dates/durations: `Intl.NumberFormat`/`Intl.DateTimeFormat` with the
  active locale; countdown digits are locale-independent by design.
- `N/A` and `‚Äì` fallbacks are **not** translated (universal per portfolio
  convention).
- CI guard: a Vitest test diffs key sets of `en.json` vs `vi.json` ‚Äî any
  missing/extra key fails the build (`13 ¬ß3`).
